# 커뮤니티 메뉴 확장 (게시판 → 커뮤니티: 게시판 + 갤러리 + 포트폴리오)

> 작성일: 2026-03-03

## Context

상위 메뉴 "게시판"을 "커뮤니티"로 변경하고 하위메뉴를 게시판, 갤러리, 포트폴리오로 확장.
- URL: `/community/*` 로 통일 (기존 `/board` → `/community/board`, 리다이렉트 처리)
- 갤러리: 이미지 카드 그리드 (작품 게시), 로그인 사용자 모두 작성 가능
- 포트폴리오: 블로그형 레이아웃, 로그인 사용자 모두 작성 가능
- SQL은 사용자에게 제공 → 수동 실행

## 사용자 결정 사항
- URL: `/community/board`, `/community/gallery`, `/community/portfolio`
- 글쓰기 권한: 로그인 사용자 모두

## Phase 1: SQL (Supabase에서 수동 실행)

### gallery 테이블
```sql
CREATE TABLE gallery (
  id BIGSERIAL PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  image_url TEXT NOT NULL,
  category TEXT DEFAULT 'artwork',
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  author_name TEXT NOT NULL,
  views INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE gallery ENABLE ROW LEVEL SECURITY;
CREATE POLICY "gallery_select" ON gallery FOR SELECT USING (true);
CREATE POLICY "gallery_insert" ON gallery FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "gallery_delete" ON gallery FOR DELETE USING (auth.uid() = user_id);

CREATE OR REPLACE FUNCTION increment_gallery_views(item_id BIGINT)
RETURNS void AS $$
BEGIN
  UPDATE gallery SET views = views + 1 WHERE id = item_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

## Phase 4: `src/utils/supabase.js` — CRUD 함수 추가 (12개)

**Gallery (8개):**

- `getGalleryItemById(id)` — 단건 조회

- `deleteGalleryItem(id)` — 삭제
- `incrementGalleryViews(id)` — RPC `increment_gallery_views`
- `getGalleryComments(galleryId)` — 댓글 목록
- `createGalleryComment(data)` — 댓글 작성
- `deleteGalleryComment(id)` — 댓글 삭제
