### 주요 문제

#### (1) 미사용 CSS 클래스 (~80~100줄)
- `.cart-*`, `.checkout-*`, `.order-*`, `.payment-*`, `.product-*` — 쇼핑몰 관련 클래스가 `responsive.css`, `dark-mode.css`에 잔존
- `shop.css`는 빈 파일

#### (2) site.css 과대 (72KB)
- 강의, 게시판, 관리자, 자료, 갤러리 등 모든 스타일이 한 파일에 집중
- 분리 권장: `lectures.css`, `boards.css`, `admin.css`, `materials.css`

#### (3) 하드코딩된 색상값
- `rgba(0, 70, 200, 0.06)` 15회 이상 반복 (→ `--primary-blue-bg` 변수 존재하나 미활용)
- `#374151` dark-mode.css에서 100회 이상 반복

#### (4) z-index 관리 부재
| 요소 | z-index | 파일 |
|------|---------|------|
| `.toast-container` | 10000 | toast.css |
| `.search-modal-overlay` | 9999 | search.css |
| `.dropdown-menu` | 999 | navbar.css |

> 계층 충돌 위험 있음

#### (5) 다크모드 대비 부족 영역
- `.badge-*` 클래스: 다크모드에서 배경이 너무 어두움
- `.auth-error`: 다크모드에서 텍스트 가독성 저하
- 코드 블록: 다크/라이트 모드 간 색상 전환 미흡

### 양호한 점
- 반응형 브레이크포인트 일관성: **9/10** (1100px → 1024px → 768px → 480px)
- CSS 변수 활용도: **7/10**

## 8. 결론

KoreaTech 사이트는 **전반적으로 안정적이며 운영 가능한 상태**입니다. 50개 라우트 모두 정상 동작하고, 배포 파이프라인도 잘 구성되어 있습니다. 주요 개선 포인트는 CSS 정리(미사용 코드 제거, 파일 분리)와 다국어 하드코딩 문자열 처리이며, AuthContext의 null 체크 2건은 잠재적 크래시를 방지하기 위해 즉시 조치를 권장합니다.
