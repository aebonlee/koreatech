# 관리자 페이지 구현 계획

## Context
Navbar의 "관리자" 버튼이 `www.dreamitbiz.com/admin`(외부)으로 이동하는 문제.
koreatech.dreamitbiz.com 자체 `/admin` 페이지를 만들어 회원목록 + 최근 접속을 확인할 수 있게 합니다.

**핵심 문제:** `user_profiles` RLS가 자기 자신만 읽기 허용 → 관리자용 읽기 정책 추가 필요.

---

## Supabase SQL (사용자 수동 실행)

```sql
-- 1. 관리자 전체 프로필 읽기 정책
CREATE POLICY "Admin can read all profiles" ON user_profiles
  FOR SELECT USING (
    auth.jwt() ->> 'email' IN ('aebon@kakao.com', 'aebon@kyonggi.ac.kr')
  );

-- 2. last_login_at 컬럼 추가
ALTER TABLE user_profiles ADD COLUMN IF NOT EXISTS last_login_at TIMESTAMPTZ;

-- 3. 최근 접속 업데이트 함수
CREATE OR REPLACE FUNCTION update_last_login(target_user_id UUID)
RETURNS void AS $$
BEGIN
  UPDATE user_profiles SET last_login_at = now() WHERE id = target_user_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

## 기능 상세

### 통계 카드 (3개)
- 전체 회원수
- 최근 7일 가입
- 최근 7일 접속

### 회원 목록 테이블
- 이름, 이메일, 가입 경로(provider), 역할, 가입일, 최근 접속
- 검색 (이름/이메일)
- provider 필터 (select)
- `board-table` 스타일 패턴 재사용

### 접근 제어
- `isAdmin` 체크 → 비관리자 접근 거부 메시지 표시
- `AuthGuard`로 라우트 보호 (로그인 필수)

### last_login_at 업데이트
- `AuthContext.loadProfile()` 에서 프로필 로드 후 `updateLastLogin(authUser.id)` 호출
- RPC `update_last_login` 사용 (SECURITY DEFINER)
