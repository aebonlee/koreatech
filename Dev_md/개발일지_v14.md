# KoreaTech 컴퓨팅 사고 교육 사이트 - 개발일지 v14

## 2026-03-03 - 커뮤니티 '웹 추천사이트' 메뉴 추가

### 개요

커뮤니티 드롭다운(게시판, 갤러리, 포트폴리오)에 4번째 하위메뉴 **'웹 추천사이트'**를 추가.
사용자가 유용한 웹 사이트 링크를 기재하고, 추천 내용을 작성할 수 있는 기능.
기존 Gallery/Portfolio 패턴을 복제하여 일관성 유지.

### 신규 기능

#### 1. 웹 추천사이트 목록 (`/community/websites`)
- 카테고리 필터: 전체 / 교육 / 개발도구 / 디자인 / AI / 참고자료
- lecture-card 스타일 3열→2열→1열 반응형 그리드
- 카드: 대표이미지(선택) + 사이트명 + URL + 추천내용 요약 + 작성자 + 조회수
- 로그인 시 "사이트 추천" 버튼 표시
- 페이지네이션

#### 2. 웹 추천사이트 상세 (`/community/websites/:id`)
- 대표 이미지 (있으면 표시)
- 사이트명, URL(클릭 가능), 작성자, 날짜, 조회수
- 추천 내용 표시
- "사이트 방문" 버튼
- 댓글 시스템
- 작성자 수정/삭제 버튼

#### 3. 사이트 등록/수정 (`/community/websites/write`, `/community/websites/edit/:id`)
- 폼: 카테고리, 사이트명, 사이트 URL(필수), 대표 이미지 URL(선택), 추천 내용
- isEdit 모드 지원

### SQL (Supabase에서 수동 실행 필요)

#### websites 테이블
```sql
CREATE TABLE websites (
  id BIGSERIAL PRIMARY KEY,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  description TEXT,
  image_url TEXT,
  category TEXT DEFAULT 'education',
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  author_name TEXT NOT NULL,
  views INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE websites ENABLE ROW LEVEL SECURITY;
CREATE POLICY "websites_select" ON websites FOR SELECT USING (true);
CREATE POLICY "websites_insert" ON websites FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "websites_update" ON websites FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "websites_delete" ON websites FOR DELETE USING (auth.uid() = user_id);

CREATE OR REPLACE FUNCTION increment_website_views(item_id BIGINT)
RETURNS void AS $$
BEGIN
  UPDATE websites SET views = views + 1 WHERE id = item_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```
